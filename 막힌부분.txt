내가 담당한 부분 : 통합검색페이지
내가 작성한 파일 : totalSearchController + com.danim.totalSearch 패키지 속 파일들 + webapp 속 view의 totalSearch.jsp + com.danim.mapper 패키지 속 totalsearch.xml

먼저 메인페이지의 통합검색창에 드롭다운으로 도(상위드롭다운)와 시(하위드롭다운), 검색유형(제목,내용,작성자)을 표시해서 사용자가 마음대로 선택할 수 있도록 하고, 
검색키워드를 작성한뒤 검색 버튼을 누르면 통합검색결과페이지로 이동하도록 서비스를 설정했습니다.

통합검색결과페이지에서는 사용자가 입력한 키워드를 바탕으로 리뷰, 축제, 먹거리게시판에서 검색한 결과를 한 화면에 따로따로 띄워줍니다.
이때 각 게시판의 검색 결과는 제목, 작성자, 작성일, 조회수만 출력되도록 설정하며, 한 페이지에 10개씩 10개의 페이지로 페이징처리하여 출력합니다.

그래서 저는 totalSearch.jsp에 간단하게 퍼블리싱을 진행했고, 이후 com.danim.totalSearch 패키지 속 VO, Mapper, Service와 com.danim.mapper 패키지 속 totalsearch.xml를
작성했습니다.

제가 막힌 부분은 통합검색창의 드롭다운 중 도와 시 드롭다운의 목록들을 데이터베이스에서 가져와서 구현하는 부분과 
게시판별로 검색 결과를 각각 페이징 처리해서 각각의 테이블로 출력하는 부분입니다.

게시판을 담당하는 팀원들이 만들때까지 기다려야하는지 아니면 따로 제가 직접 구현해야하는지, 직접 구현한다면 어떻게 해야할지, 
그리고 게시판을 담당하는 팀원들이 구현한 코드와 제가 만든 코드가 충돌이 있지는 않을지 걱정입니다.
